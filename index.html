Explore the three phases of our System Analysis & Design project

1
Phase One
  • Select an appropriate SDLC Approach (plan-driven or agile), explain it, and discuss why you chose it.
  • Scheduling & Planning: Gantt Chart, Network, WBS, Task Usage, Resources, RBS, Resource Usage.
  • Discuss how to deal with the client, gather requirements, and get the most suitable feedback (Requirement Gathering).
2
Phase Two
  • Requirement Deterministic (Functional & Non-Functional)
  • Business Model
  • Data Flow Diagram (DFD)
  • Entity Relationship Diagram (ERD)
  • Database Schema
  • Use Case Diagram
  • Use Case Scenario
3
Phase Three
  • Activity Diagram
  • Class Diagram
  • Sequence Diagram
  • Architecture View (Logical View – Physical View – Deployment View)
  • State Machine Diagram
  • UI Design
